02 / 15 · 문제 정의
한 줄 정의

선언적 manifest를 해석해 설치 마법사 + 후처리까지 단일 흐름으로 처리하는 Windows 통합 인스톨러.

앱 팀은 manifest.json 하나만 작성. 사용자는 인스톨러 하나만 설치하면 됨. UAD Connect / Native Access 같은 모델을 Samsung 앱 + 에이전트에 적용.

참고 모델

이미 검증된 패턴

개별 인스톨러를 통합 호스트로 대체하는 모델은 프로 오디오·플러그인 생태계에서 10년 넘게 검증됨.

UAD Connect
Universal Audio. 수십 개 DSP 플러그인을 단일 catalog UI에서 설치/업데이트/제거. 라이선스도 통합 관리.
Native Access
Native Instruments. 모든 KOMPLETE 제품의 카탈로그 + 다운로드 + 마법사 + 위치 이동을 한 앱이 처리.
iLok License Manager
Pace. 수백 개 plugin 라이선스 발급/이전을 단일 클라이언트로 통합.

VDX Installer는 같은 패턴을 Samsung 사내 앱 + AI 에이전트에 적용한다.

04 / 15 · 영감
핵심 원리

선언적 manifest + 통합 엔진

각 앱의 설치 방식을 코드로 적지 않고 JSON 한 장으로 선언. 호스트는 그것만 해석.

개별 앱 인스톨러 EXE 불필요. 모든 후처리(레지스트리/PATH/단축키/exec)는 manifest 액션으로 표현.

06 / 15 · 아키텍처
manifest.json

설치를 코드가 아니라 "선언"으로 표현

// 발췌
{
  "id": "com.samsung.vdx.exampleapp",
  "version": "1.2.3",
  "name": { "ko": "예제 앱", "default": "Example" },
  "installScope": { "supports": ["user"], "default": "user" },

  // 마법사 — 6가지 step 타입 (license, path, checkboxGroup,
  //                       select, text, summary)
  "wizard": [
    { "type": "license", "id": "eula", "fileFromPayload": "EULA.txt" },
    { "type": "path", "id": "installPath", "default": "{LocalAppData}/…" },
    { "type": "checkboxGroup", "id": "components", "items": [/* … */] },
    { "type": "summary" }
  ],

  // 후처리 — 4가지 액션 (shortcut, registry, envPath, exec)
  "postInstall": [
    { "type": "shortcut", "where": "desktop",
      "target": "{{installPath}}/x.exe", "name": "Example" },
    { "type": "registry", "hive": "HKCU",
      "key": "Software\\Samsung\\X",
      "values": { "InstallPath": { "type": "REG_SZ",
                                "data": "{{installPath}}" } } },
    { "type": "exec", "cmd": "{{installPath}}/post.exe",
      "timeoutSec": 30,
      "allowedHashes": ["sha256:abc…"] }
  ]
}
Wizard step 6 종
license · path · checkboxGroup · select · text · summary. 그 외 타입은 schema 거부.
PostInstall action 4 종
shortcut, registry, envPath, exec. 임의 셸 실행 금지. exec는 sha256 화이트리스트 필수.
{{var}} + when 표현식
wizard 답변 → 액션에 주입. when은 한정된 grammar(== != && || !) 의 hand-written parser. eval 금지.
strict mode + 컨트롤 문자 차단
모든 manifest 문자열은 NUL/제어문자 거부 (Win32 API 잘림 방지). shortcut name은 추가로 path-separator 차단 (디렉토리 escape 방지).
07 / 15 · manifest 구조
설치 마법사

모든 앱이 같은 흐름으로 설치됨

각 앱은 wizard step만 선언. UI는 호스트가 일관되게 렌더 → 사용자는 항상 같은 모양의 화면을 본다.

업데이트 시 답변 재사용
새 manifest의 step 중 id+type 같은 항목은 묻지 않고 재사용. 새/변경된 step만 표시. summary만 남으면 마법사 통째로 스킵.
runtime 검증
각 step의 답변은 zod schema + regex로 검증. 빈 path / 잘못된 텍스트 / 필수 체크박스 미선택 → 즉시 에러.
i18n ready
label/option label은 { ko, en, default } 형태. 시스템 로캘 따라 자동 선택, fallback 보장.
08 / 15 · 마법사
신뢰성

트랜잭션 + journal 기반 원자적 설치

설치 도중 실패 / 강제 종료 / 정전 어떤 상황에서도 시스템을 원래 상태로 복구.

stepOrdinal 정렬
journal 외부 변조에도 강함. .steps[]가 섞여 있어도 stepOrdinal 내림차순으로 정확히 역재생.
크래시 복구
호스트가 강제 종료되면 다음 부팅 때 journal/ 디렉토리에서 미완료 트랜잭션 자동 감지 → 사용자 동의 후 정리.
Per-app advisory lock
locks/<app-id>.lock 으로 동일 앱 동시 설치 차단. 죽은 PID 자동 회수.
09 / 15 · 트랜잭션
보안

HSM → 코드사인 + Ed25519 → 호스트

두 개의 독립된 신뢰 경로. 어느 한쪽이 깨져도 다른 쪽이 catalog/manifest/payload 변조를 잡아냄.

exec hash 화이트리스트
postInstall.exec 는 sha256 매칭만 실행. 임의 셸·파라미터 금지.
Sideload 경로 allowlist
sideload:open은 picker가 발급한 경로만 1회 허용.
CSP + sandboxed renderer
production: connect-src 'none'. renderer는 IPC 외 어떤 네트워크도 못 함.
로그 redaction
GH 토큰, Bearer, hex key, Slack/AWS 키, 홈 경로 자동 마스킹.

13 / 15 · 퍼포먼스
앱 / 에이전트 팀 워크플로

vdx-pack CLI + AI 어시스턴트가 패키징을 처리

앱 팀은 manifest.json만 작성. 검증·해시·서명·zip 빌드는 CLI가, 그 CLI를 호출하는 루프는 Cursor / Cline 룰 파일이.

# 검증 — schema + 의미 검사
$ vdx-pack validate manifest.json --payload ./payload
✓ com.samsung.vdx.example v1.2.3 valid

# 해시 — exec.allowedHashes 에 박을 값
$ vdx-pack hash ./payload/post.exe
sha256:98ea6e4f216f2fb4b69fff9b3a44842c…

# 빌드 + 서명 — 서명되지 않은 패키지는 dev 모드만
$ vdx-pack build \
    --manifest manifest.json \
    --payload ./payload \
    --out my-agent-1.0.0.vdxpkg \
    --key ~/.vdx/private-signing.bin
✓ built my-agent-1.0.0.vdxpkg (412.3 KiB)
  payload sha256: 2365ba39…
  signed: YES

# --json 으로 출력 → AI 어시스턴트가 파싱
$ vdx-pack validate manifest.json --json | jq
.cursor/rules/vdx-packaging.mdc
Cursor가 manifest 편집 후 자동으로 vdx-pack validate --json 돌리고 결과로 self-correct하는 루프 룰.
.clinerules/vdx-packaging.md
Cline 자율 모드용 동일 내용. AI는 manifest 의도만 받으면 검증 통과까지 알아서.
샘플 fixtures
npm run gen:agent-catalog 한 줄로 catalog.json + 2개 에이전트 .vdxpkg + sha256 sidecar + README 생성. GitHub Pages 브랜치에 그대로 복사.
